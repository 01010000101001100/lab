A simple responsive website

Resize this website to see how it works. The simplest code required to make your website responsive:

body {
  max-width: 680px;
  margin: 0 auto;
}

That’s it. The content will stay centered and resize nicely on small screens.

You could stop there, but let’s build on that and improve usability.

Mobile Safari issue

First, let’s fix an issue with mobile Safari that renders the website in 980px (more on viewports) and might give you some headeaches with styling your layout. To fix that, we add a meta tag in HTML that basically says one screen pixel equals one viewport pixel:

<meta name="viewport" content="width=device-width"/>

Responsive images and videos

Let’s add some CSS code to also make our images and videos responsive:

img, embed, object, video {
  max-width: 100%;
}

Small screens

We could stop here but let’s further improve the layout on small screens. I define small screens as devices that have a resolution smaller than 767px. I chose this number for pixels so that I exclude the iPad, which has a width of 768px, from the small screens (i.e. mobile) category.

For small screens we make the font smaller, the line height smaller, add some horizontal spacing to content, and increase the contrast of the text. We also make the main heading of the website smaller:

@media screen and (max-width: 767px) {
  body {
    font-size: 1em; /* if desktop body size is not default (mine is 18px) */
    line-height: 1.6em;
    color: #111;
    width: 95%;
  }

  h1 {
    font-size: 2em;
  }
}

So, for a not-so-simple responsive website you add a total of 19 lines of code. Not bad.

Large screens

For the sake of this tutorial, I also added a large screen media query that increases the font size of everything. The pixel width was pretty much a random number not exceeding 1440px (biggest resolution on non-retina MacBook Pros/Airs):

@media screen and (min-width: 1375px) {
  body {
    font-size: 1.33333333em;
  }
}
